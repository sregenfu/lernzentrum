Tag 2 im Detail: Scales und Achsen

Heute übersetzt du Datenwerte sauber in Pixelpositionen. Mit der Margin Convention zeichnest du Achsen und Balken in einem inneren Zeichenbereich, damit Beschriftungen genug Platz haben.

Aufgabe 1 - Balkendiagramm mit Scales aufbauen

Nutze in main_Tage2.js ein Array aus Objekten und erzeuge ein Diagramm mit kategorischer X-Achse und numerischer Y-Achse.

Startcode (in main_Tage2.js einfügen):

// Tag 2 - Scales und Achsen

const data = [

  { name: "Mo", value: 12 },

  { name: "Di", value: 18 },

  { name: "Mi", value: 9 },

  { name: "Do", value: 24 },

  { name: "Fr", value: 15 }

];



const width = 760;

const height = 420;

const margin = { top: 24, right: 20, bottom: 52, left: 56 };



const innerWidth = width - margin.left - margin.right;

const innerHeight = height - margin.top - margin.bottom;



const svg = d3.select("#chart")

  .attr("viewBox", `0 0 ${width} ${height}`);



const g = svg.append("g")

  .attr("transform", `translate(${margin.left},${margin.top})`);



const x = d3.scaleBand()

  .domain(data.map(d => d.name))

  .range([0, innerWidth])

  .padding(0.2);



const y = d3.scaleLinear()

  .domain([0, d3.max(data, d => d.value)])

  .nice()

  .range([innerHeight, 0]);



g.selectAll("rect")

  .data(data)

  .join("rect")

  .attr("x", d => x(d.name))

  .attr("y", d => y(d.value))

  .attr("width", x.bandwidth())

  .attr("height", d => innerHeight - y(d.value))

  .attr("fill", "#2563eb");



g.append("g")

  .attr("transform", `translate(0,${innerHeight})`)

  .call(d3.axisBottom(x));



g.append("g")

  .call(d3.axisLeft(y).ticks(6));

Was du dabei lernst

  • scaleBand() für Kategorien auf der X-Achse
  • scaleLinear() für numerische Werte auf der Y-Achse
  • .range([innerHeight, 0]) weil SVG-Y nach unten wächst
  • axisBottom(x) und axisLeft(y) für Achsen
  • Warum die Margin Convention die Lesbarkeit verbessert

Aufgabe 2 - Selbst ausprobieren

  1. Ersetze die Tagesdaten durch deine echten Lernwerte.
  2. Füge eine Y-Achsen-Beschriftung hinzu (z. B. Minuten).
  3. Experimentiere mit .padding(0.1) und .padding(0.35).
Profi-Tipp: Wenn Balken falsch herum aussehen, prüfe zuerst die Y-Scale-Range und die Balkenhöhe innerHeight - y(d.value).
