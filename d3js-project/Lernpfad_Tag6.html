Tag 6: Echte Daten einlesen

Heute lädst du eine CSV-Datei statt hardcodierter Arrays. Schwerpunkt: d3.csv(), d3.json(), Typkonvertierung, .then() / .catch().

Für diese Seite wird die Datei data/lernzeiten.csv benötigt. Der lokale Server muss dafür im Projekt-Root laufen.

Tag 6 im Detail: Echte Daten einlesen

Hardcodierte Arrays reichen schnell nicht mehr aus. Mit d3.csv() und d3.json() liest du Daten asynchron aus Dateien  wie in echten Projekten.

Aufgabe 1 - CSV-Datei laden und visualisieren

Lege eine Datei data/lernzeiten.csv an und lade sie mit d3.csv(). Achte auf Typkonvertierung: CSV-Werte sind immer Strings.

CSV-Beispieldatei (data/lernzeiten.csv):

tag,minuten

Mo,45

Di,30

Mi,60

Do,20

Fr,50

Startcode (in main_Tage6.js einfügen):

// Tag 6 - Echte Daten mit d3.csv()

d3.csv("data/lernzeiten.csv", d => ({

  tag: d.tag,

  minuten: +d.minuten       // + konvertiert String zu Number

})).then(data => {



  const width = 760, height = 420;

  const margin = { top: 24, right: 20, bottom: 52, left: 64 };

  const innerWidth = width - margin.left - margin.right;

  const innerHeight = height - margin.top - margin.bottom;



  const svg = d3.select("#chart").attr("viewBox", `0 0 ${width} ${height}`);

  const g = svg.append("g").attr("transform", `translate(${margin.left},${margin.top})`);



  const x = d3.scaleBand().domain(data.map(d => d.tag)).range([0, innerWidth]).padding(0.2);

  const y = d3.scaleLinear().domain([0, d3.max(data, d => d.minuten)]).nice().range([innerHeight, 0]);



  g.append("g").attr("transform", `translate(0,${innerHeight})`).call(d3.axisBottom(x));

  g.append("g").call(d3.axisLeft(y).ticks(6));



  // Y-Achsen-Beschriftung

  g.append("text")

    .attr("transform", "rotate(-90)")

    .attr("x", -innerHeight / 2)

    .attr("y", -48)

    .attr("text-anchor", "middle")

    .attr("fill", "#64748b")

    .attr("font-size", "13px")

    .text("Minuten");



  g.selectAll("rect")

    .data(data)

    .join("rect")

    .attr("x", d => x(d.tag))

    .attr("y", d => y(d.minuten))

    .attr("width", x.bandwidth())

    .attr("height", d => innerHeight - y(d.minuten))

    .attr("fill", "#2563eb");



}).catch(err => console.error("Fehler beim Laden:", err));

Was du dabei lernst

  • d3.csv(url, rowFn) - Datei laden + Zeilen umwandeln in einem Schritt
  • +d.minuten - Unärer Plus: String zu Number
  • .then(data => { ... }) - Promise-basierter Async-Flow
  • .catch(err => ...) - Fehlerbehandlung wenn Datei fehlt
  • Warum der Server laufen muss (CORS-Restriction bei lokalen Dateien)

Aufgabe 2 - Selbst ausprobieren

  1. Erweitere die CSV um eine Spalte kategorie und färbe die Balken je Kategorie.
  2. Lade stattdessen eine JSON-Datei mit d3.json().
  3. Zeige eine Fehlermeldung im HTML, wenn die Datei nicht gefunden wird.
Profi-Tipp: Starte immer mit console.log(data) direkt nach dem Laden, um die Datenstruktur zu prüfen, bevor du visualisierst. Viele Fehler entstehen durch falsche Feldnamen oder fehlende Typkonvertierung.
