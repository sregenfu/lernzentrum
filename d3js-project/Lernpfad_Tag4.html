Tag 4: Transitionen und Animation

Heute animierst du Balken beim Laden und beim Datenwechsel. Schwerpunkt: transition(), duration(), delay(), ease().

Tag 4 im Detail: Transitionen und Animation

Animationen machen Veränderungen im Diagramm sichtbar und helfen dem Auge, Übergänge zu verfolgen. D3-Transitionen laufen auf CSS-Übergängen auf und sind leichtgewichtig.

Aufgabe 1 - Balken mit Transition einblenden

Lass die Balken beim ersten Laden von Höhe 0 auf ihren Zielwert wachsen und beim Datenwechsel sanft von alt nach neu übergehen.

Startcode (in main_Tage4.js einfügen):

// Tag 4 - Transitionen

const width = 760, height = 420;

const margin = { top: 24, right: 20, bottom: 52, left: 56 };

const innerWidth = width - margin.left - margin.right;

const innerHeight = height - margin.top - margin.bottom;



const svg = d3.select("#chart")

  .attr("viewBox", `0 0 ${width} ${height}`);



const g = svg.append("g")

  .attr("transform", `translate(${margin.left},${margin.top})`);



const categories = ["Mo", "Di", "Mi", "Do", "Fr"];



const x = d3.scaleBand().domain(categories).range([0, innerWidth]).padding(0.2);

const y = d3.scaleLinear().domain([0, 50]).range([innerHeight, 0]);



g.append("g").attr("transform", `translate(0,${innerHeight})`).call(d3.axisBottom(x));

g.append("g").call(d3.axisLeft(y).ticks(6));



function update(data) {

  g.selectAll("rect")

    .data(data, d => d.name)

    .join(

      enter => enter.append("rect")

        .attr("x", d => x(d.name))

        .attr("y", innerHeight)           // Startposition unten

        .attr("width", x.bandwidth())

        .attr("height", 0)               // Starthoehe 0

        .attr("fill", "#2563eb"),

      update => update,

      exit => exit.remove()

    )

    .transition().duration(600).ease(d3.easeCubicOut)

    .attr("y", d => y(d.value))

    .attr("height", d => innerHeight - y(d.value));

}



const data = categories.map(name => ({ name, value: Math.round(Math.random() * 40 + 5) }));

update(data);



d3.select("body").insert("button", "svg")

  .text("Neue Werte animieren")

  .style("margin", "12px 0")

  .on("click", () => {

    const newData = categories.map(name => ({ name, value: Math.round(Math.random() * 40 + 5) }));

    update(newData);

  });

Was du dabei lernst

  • .transition().duration(600) - Übergang mit 600ms Dauer
  • .ease(d3.easeCubicOut) - Easing-Funktion für natürliche Bewegung
  • Enter-Phase getrennt mit Startposition y = innerHeight, height = 0
  • .delay((d, i) => i * 60) - gestaffelter Einblendeffekt je Index

Aufgabe 2 - Selbst ausprobieren

  1. Füge .delay((d, i) => i * 80) hinzu  Balken erscheinen versetzt.
  2. Ändere easeCubicOut auf easeBounce und beobachte den Unterschied.
  3. Animiere auch die Farbe über .attr("fill", ...) in der Transition.
Profi-Tipp: Übergänge laufen immer auf interpolierbaren Werten. Versuche nie, font-family oder display zu animieren  das funktioniert nicht. Sichere Kandidaten: numerische Attribute und Farben.
