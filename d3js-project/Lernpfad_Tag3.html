Tag 3: Data Join richtig verstehen

Heute baust du ein Diagramm, das auf Knopfdruck neue Daten lädt und sich sauber aktualisiert. Schwerpunkt: .join(), key-Funktionen, Enter / Update / Exit.

Tag 3 im Detail: Data Join richtig verstehen

D3 unterscheidet drei Fälle beim Aktualisieren von Daten: Enter (neue Elemente), Update (bestehende Elemente) und Exit (zu löschende Elemente). selection.join() fasst alle drei Fälle zusammen und ist heute der empfohlene Weg.

Aufgabe 1 - Diagramm dynamisch aktualisieren

Füge in main_Tage3.js einen Button ein, der bei jedem Klick einen neuen zufälligen Datensatz lädt und das Diagramm aktualisiert.

Startcode (in main_Tage3.js einfügen):

// Tag 3 - Data Join mit .join()

const width = 760, height = 420;

const margin = { top: 24, right: 20, bottom: 52, left: 56 };

const innerWidth = width - margin.left - margin.right;

const innerHeight = height - margin.top - margin.bottom;



const svg = d3.select("#chart")

  .attr("viewBox", `0 0 ${width} ${height}`);



const g = svg.append("g")

  .attr("transform", `translate(${margin.left},${margin.top})`);



const categories = ["Mo", "Di", "Mi", "Do", "Fr", "Sa", "So"];



function randomData() {

  return categories.map(name => ({ name, value: Math.round(Math.random() * 40 + 5) }));

}



const x = d3.scaleBand().domain(categories).range([0, innerWidth]).padding(0.2);

const y = d3.scaleLinear().range([innerHeight, 0]);



const xAxis = g.append("g").attr("transform", `translate(0,${innerHeight})`);

const yAxis = g.append("g");



function update(data) {

  y.domain([0, d3.max(data, d => d.value)]).nice();

  xAxis.call(d3.axisBottom(x));

  yAxis.call(d3.axisLeft(y).ticks(6));



  g.selectAll("rect")

    .data(data, d => d.name)        // key-Funktion: Identität per Name

    .join("rect")                   // enter + update + exit in einem

    .attr("x", d => x(d.name))

    .attr("y", d => y(d.value))

    .attr("width", x.bandwidth())

    .attr("height", d => innerHeight - y(d.value))

    .attr("fill", "#2563eb");

}



update(randomData());



// Button zum Auslösen

d3.select("body").insert("button", "svg")

  .text("Neue Daten laden")

  .style("margin", "12px 0")

  .on("click", () => update(randomData()));

Was du dabei lernst

  • .data(data, d => d.name) - key-Funktion verhindert unnötiges Neuzeichnen
  • .join("rect") - kombiniert Enter, Update und Exit
  • Warum key-Funktionen bei animierten Updates entscheidend sind
  • Skalierungen per .domain() neu setzen ohne die Achsen neu zu erstellen

Aufgabe 2 - Selbst ausprobieren

  1. Baue einen zweiten Button, der nur 3 Kategorien anzeigt  beobachte den Exit.
  2. Füge eine Farbe hinzu, die sich je nach Wert ändert (hoch = grün, niedrig = rot).
  3. Entferne die key-Funktion und schau, was sich beim Update ändert.
Profi-Tipp: Debugge den Data Join mit console.log(selection.data()) nach jeder Bindung, um zu sehen, welche Daten welchen DOM-Elementen zugeordnet sind.
